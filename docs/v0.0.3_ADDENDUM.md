# v0.0.3 / v0.0.31 Addendum — Package Supply Chain Analysis

This addendum extends the original build prompt and design documents with the package IOC types.

## New IOC Types

### `package` — Prefixed ecosystem:name
**Input format**: `ecosystem:name` (e.g., `npm:postmark-mcp`, `pypi:requessts`, `rhel:openssl`)

### `package_multi` — Bare package name (multi-ecosystem scan)
**Input format**: A bare package name with no prefix (e.g., `traceroute`, `express`, `lodash`)

When a bare name is submitted, the agent scans it across **10 major language ecosystems simultaneously**: npm, PyPI, crates.io, Go, Maven, NuGet, RubyGems, Packagist, Pub, Hex.

The report shows a per-ecosystem breakdown with results for each.

## Data Sources (No API Keys Required)

### OSV.dev (Google Open Source Vulnerabilities)
- **Endpoint**: POST `https://api.osv.dev/v1/query`
- **Auth**: None required
- **Body**: `{"package": {"name": "<n>", "ecosystem": "<ecosystem>"}}`
- **Multi-scan**: For bare package names, queries all 10 MULTI_SCAN_ECOSYSTEMS in parallel

### Package Registry Metadata (npm + PyPI only)
- **npm**: GET `https://registry.npmjs.org/{package}`
- **PyPI**: GET `https://pypi.org/pypi/{package}/json`
- **Auth**: None required

## Weight Sets

```python
PACKAGE_WEIGHTS = {"osv": 0.60, "registry": 0.40}          # Prefixed packages
PACKAGE_MULTI_WEIGHTS = {"osv_multi": 1.00}                 # Bare package names
```
